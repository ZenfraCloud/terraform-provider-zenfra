# Zenfra Provider

The Zenfra provider enables you to manage [Zenfra](https://zenfra.cloud) resources using Terraform or OpenTofu. It supports managing spaces, stacks, worker pools, configuration bundles, API tokens, VCS integrations, and more.

## Authentication

The provider requires an API token to authenticate with the Zenfra API. You can provide it in two ways:

### Environment variable (recommended)

```shell
export ZENFRA_API_TOKEN="your-api-token"
```

```terraform
provider "zenfra" {}
```

### Provider configuration

```terraform
provider "zenfra" {
  api_token = var.zenfra_api_token
}
```

## Custom endpoint

By default, the provider connects to `https://api.zenfra.cloud`. You can override this for self-hosted or development environments:

```terraform
provider "zenfra" {
  endpoint = "https://api.your-instance.example.com"
}
```

Or via environment variable:

```shell
export ZENFRA_API_ENDPOINT="https://api.your-instance.example.com"
```

## Example Usage

```terraform
# ABOUTME: Example Terraform configuration demonstrating Zenfra provider setup.
# ABOUTME: Shows required_providers block and provider configuration with endpoint and api_token.

terraform {
  required_providers {
    zenfra = {
      source = "registry.terraform.io/ZenfraCloud/zenfra"
    }
  }
}

provider "zenfra" {
  # endpoint  = "https://api.zenfra.cloud"  # Optional: defaults to https://api.zenfra.cloud
  # api_token = "your-api-token"          # Or set ZENFRA_API_TOKEN env var
}
```
