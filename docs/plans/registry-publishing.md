## Phase 1: GPG Key (CI-only context)

Generate the key on a clean machine (or ephemeral VM/container), export to GitHub Secrets and org secret manager, then delete the local copy. The private key should only ever live in GitHub's encrypted secrets store and your secure backup — never on a developer laptop.

### Step 1 — Generate RSA key on a clean machine

```bash
# Use a temporary GNUPGHOME so nothing touches your personal keyring
export GNUPGHOME=$(mktemp -d)

gpg --batch --gen-key <<'GPG_EOF'
%no-protection
Key-Type: RSA
Key-Length: 4096
Name-Real: Zenfra
Name-Email: security@zenfra.cloud
Expire-Date: 0
%commit
GPG_EOF
```

- **RSA 4096** — both registries reject ECC/Ed25519
- **No expiry** — avoids release failures from expired keys; rotate proactively if needed
- **Org identity** (`Zenfra <security@zenfra.cloud>`) — not personal; survives team changes
- **`%no-protection`** — no passphrase on the key itself (simpler CI setup; the key is protected by GitHub Secrets encryption at rest). If your security policy requires a passphrase, remove this line and set one interactively, then store it in `GPG_PASSPHRASE` secret.

### Step 2 — Export both keys

```bash
# Get the fingerprint
FINGERPRINT=$(gpg --list-keys --keyid-format long --with-colons | grep fpr | head -1 | cut -d: -f10)
echo "Fingerprint: $FINGERPRINT"

# Export public key (needed for both registries)
gpg --armor --export "$FINGERPRINT" > zenfra-gpg-public.asc

# Export private key (needed for GitHub Actions)
gpg --armor --export-secret-keys "$FINGERPRINT" > zenfra-gpg-private.asc
```

### Step 3 — Store the keys

| What | Where | How |
|------|-------|-----|
| **Private key** (`zenfra-gpg-private.asc`) | GitHub repo secret `GPG_PRIVATE_KEY` | Paste full contents including `-----BEGIN PGP PRIVATE KEY BLOCK-----` |
| **Passphrase** | GitHub repo secret `GPG_PASSPHRASE` | Leave empty if `%no-protection` was used |
| **Public key** (`zenfra-gpg-public.asc`) | Terraform Registry — User Settings > Signing Keys | Paste after signing in (Phase 3, Step 16) |
| **Public key** (`zenfra-gpg-public.asc`) | OpenTofu Registry — GitHub issue form | Submit via web UI (Phase 4, Step 20) |
| **Both keys** | Org secret manager (1Password / Vault / etc.) | Backup for recovery if GitHub secrets are lost |

To add GitHub secrets: `ZenfraCloud/terraform-provider-zenfra` > Settings > Secrets and variables > Actions > New repository secret.

`GPG_FINGERPRINT` is **not** a secret — the release workflow extracts it automatically from the imported key via `crazy-max/ghaction-import-gpg`.

### Step 4 — Destroy local copies

```bash
# Wipe the temporary keyring
rm -rf "$GNUPGHOME"

# Delete exported files (after confirming they're saved in GitHub Secrets + secret manager)
rm zenfra-gpg-private.asc zenfra-gpg-public.asc

# Clear the env var
unset GNUPGHOME
```

After this step, the private key exists only in GitHub Secrets and your org secret manager. No developer machine has a copy.

### Step 11 — Create `.github/workflows/release.yml`

Triggered on `v*` tag push. Steps:

1. Checkout with full history (`git fetch --prune --unshallow` — needed for changelog)
2. Setup Go (from `go.mod`)
3. Import GPG key using `crazy-max/ghaction-import-gpg@v7` (reads `GPG_PRIVATE_KEY` and `GPG_PASSPHRASE` secrets, outputs fingerprint)
4. Run GoReleaser (`goreleaser release --clean`) with fingerprint and `GITHUB_TOKEN`

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  goreleaser:
    name: Release
    runs-on: ubuntu-latest
    permissions:
      contents: write

    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Unshallow
        run: git fetch --prune --unshallow

      - name: Set up Go
        uses: actions/setup-go@v6
        with:
          go-version-file: go.mod

      - name: Import GPG key
        uses: crazy-max/ghaction-import-gpg@v7
        id: import_gpg
        with:
          gpg_private_key: ${{ secrets.GPG_PRIVATE_KEY }}
          passphrase: ${{ secrets.GPG_PASSPHRASE }}

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v7
        with:
          version: "~> v2"
          args: release --clean
        env:
          GPG_FINGERPRINT: ${{ steps.import_gpg.outputs.fingerprint }}
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Step 12 — Create `.github/workflows/check-documentation.yml`

Runs on pushes and PRs to main. Regenerates docs and fails if they are stale.

```yaml
name: Documentation

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  check-documentation:
    name: Check documentation is up to date
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v6

      - name: Install Go
        uses: actions/setup-go@v6
        with:
          go-version-file: go.mod

      - name: Install Terraform
        uses: hashicorp/setup-terraform@v4
        with:
          terraform_version: "~1"
          terraform_wrapper: false

      - name: Generate documentation
        run: cd tools && go generate ./...

      - name: Verify no changes
        uses: tj-actions/verify-changed-files@v20
        id: verify-changed-files
        with:
          files: docs

      - name: Fail if docs are stale
        if: steps.verify-changed-files.outputs.files_changed == 'true'
        run: |
          echo "docs/ is out of date. Run 'cd tools && go generate ./...' and commit."
          git diff
          exit 1
```

## Phase 3: Terraform Registry (manual)

### Step 15 — Sign in

Go to registry.terraform.io/sign-in. Authenticate with the **ZenfraCloud** GitHub org (need admin access to the repo).

### Step 16 — Add GPG public key

User Settings > Signing Keys > paste contents of `zenfra-gpg-public.asc`.

### Step 17 — Publish the provider

Publish > Provider > select `ZenfraCloud/terraform-provider-zenfra`. The registry installs a webhook on the repo.

### Step 18 — Create first release

```bash
git tag v0.1.0
git push origin v0.1.0
```

The `release.yml` workflow fires, GoReleaser builds + signs + publishes the release, the registry webhook picks it up automatically.

### Step 19 — Verify

Visit `registry.terraform.io/providers/zenfra/zenfra/latest`. Should show docs, download links, and install instructions.

## Phase 4: OpenTofu Registry (manual)

### Step 20 — Submit GPG key

Go to the OpenTofu registry repo issue form for submitting a provider key:
`https://github.com/opentofu/registry/issues/new?template=provider_key.yml`

**Must use the web form UI** — `gh` CLI and API submissions are not processed.

Fill in:
- Namespace: `zenfra`
- ASCII-armored public key

### Step 21 — Submit provider

Go to the OpenTofu registry repo issue form for submitting a provider:
`https://github.com/opentofu/registry/issues/new?template=provider.yml`

**Must use the web form UI.**

Fill in the repo: `ZenfraCloud/terraform-provider-zenfra`.

### Step 22 — Wait for approval

The OpenTofu team reviews and approves. After approval, the registry auto-detects existing and future git tags.
